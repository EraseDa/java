자동로그인
사이트에 접속하면 (어떤 URL이든 상관없이) 자동로그인인터셉터가 동작
 => 세션에 회원 정보가 있는지 확인(로그인 된 상태인지 확인)
     => 로그인 되었으면 건너뜀(자동 로그인 필요 없음)
     => 로그인 안되어있으면 자동 로그인을 시도
          => 특정의 이름의 쿠키가 있는지 확인
               => 없음. 자동로그인을 원하지 않음
               => 있음. 자동로그인을 원함 => 자동 로그인 시도
                   => 쿠키에 값을 가져옴
                   => DB와 일치하는 회원 정보가 있는지 확인(자세한건 다시 설명)
                       => 있으면 세션에 저장
  
사전 작업
 - 회원 테이블 수정(속성 추가)
   - 세션 아이디(쿠키에 저장) : me_session_id
   - 세션 만료시간 : me_session_limit

 - 로그인인터셉터
   - 로그인을 성공하면 자동 로그인을 할건지 확인
     =>자동로그인이 체크가 되어있으면 
         - 쿠키를 생성 : 이름 - A, 값 - 세션아이디
         - 만료시간을 계산 : s단위
         - 쿠키의 경로와 만료시간을 설정
         - response에 쿠키를 추가
         - 회원 정보에 세션아이디, 세션 만료시간을 추가함
         - 수정된 회원 정보를 이용하여 DB에 업데이트

 - 로그아웃
   - DB에서 세션만료시간을 null로 수정

 - login.jsp
   - 자동 로그인체크박스 추가
 - MemberVO 필드 추가
   - me_session_id, me_session_limit, autoLogin
